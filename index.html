브랜드 가치를 숫자로 — 데이터 기반 브랜드 지표 체계 구축

앤서스랩코리아 — GGPoker & WSOP
# Brand Metrics# KPI 모니터링 # Social Listening# BigQuery · Snowflake
전략적 판단
"데이터가 있어도 믿을 수 없으면 의사결정에 쓸 수 없습니다." 플랫폼 전환기에 수치 신뢰성이 무너지면 마케팅 판단 자체가 흔들린다는 문제를 인식하고, 데이터 정합성 확보를 전략 실행의 선결 조건으로 정의했습니다. 단순 리포팅 자동화가 아니라, 이상 징후를 자동으로 감지해 마케터가 "이상하다"고 느끼기 전에 시스템이 먼저 포착하는 구조를 목표로 설정했습니다.
실행
글로벌 16개 리전 × 7개 KPI를 단일 파이프라인으로 통합. 플랫폼 전환(BigQuery→Snowflake) 중에도 데이터 공백 없이 리포팅이 유지되도록 포맷 변환 로직과 결측 처리를 직접 설계했습니다. 포커 스테이크 그룹을 비즈니스 로직에 맞게 6개 구간으로 재분류하고, Z-score 기반 이상탐지로 수동 모니터링을 시스템이 대체하는 구조를 구축했습니다.
Action
BigQuery FORMAT_DATE → Snowflake TO_CHAR 포맷을 마이그레이션이 다를 중 미리 처리하는 데이터 파이프라인 구축. NULLIF 처리로 이중 오류 0%로 낮추고 데이터 클리닝 이슈 해결.
CASE WHEN으로 이 비정형 Stake 그룹 Freeroll → Super High Stakes 6개 구간 + Other로 비즈니스 로직 재분류.
Topic x Metric 단위 그룹핑 및 Z-score 기반 이상 탐지 로직 설계로 시계열 이상치 자동 탐지 체계 구축.
결과 · 영향
플랫폼 전환기에도 데이터 공백 없이 리포팅 유지. Z-score 이상탐지 도입으로 담당자가 수동 확인하기 전에 시스템이 먼저 이상 성과를 포착하는 체계 구축. 이로써 마케팅팀이 "숫자를 의심하는 시간" 대신 "전략을 논의하는 시간"을 확보하게 됐습니다.

"리뷰 회의 전날 밤을 수작업으로 보내는 팀은 전략이 아니라 집계를 하고 있는 것입니다." 4개 소스에 분산된 데이터를 매번 수작업으로 취합하는 구조가 마케팅팀의 전략적 판단 속도를 저하시키고 있다고 진단했습니다. 단순 자동화가 아니라, "이상 징후가 생기면 시스템이 먼저 알려주는" 대시보드를 목표로 설정해 팀이 데이터를 읽는 데가 아닌 전략을 논의하는 데 시간을 쓸 수 있는 구조를 기획했습니다.
실행
4개 소스를 단일 파이프라인으로 통합하고, 16개 리전 × 7개 KPI를 한 화면에서 비교할 수 있는 인터랙티브 대시보드를 직접 구현했습니다. SPIKE·PLUNGE·ABNORMAL 3단계 이상 플래그 시스템을 설계해 담당자가 매번 수치를 훑지 않아도 이상 항목이 자동 표시되도록 했습니다.
Action

Region Comparison 색상 팔레트 기존 7→16으로 확장, 전체 리전 데이터 비교 지원. Rest Of World 비중을 포함 국가 몇 일 및 국가 파이 구성, 영역 레이블 설정.
Google Apps Script 단 환경 내에서 백엔드(데이터처리)와 프론트엔드(시각화)를 모두 구현.

수작업 집계 → 자동화 대시보드 전환으로 팀 리뷰 준비 시간을 대폭 단축. 16개 리전 × 7개 KPI를 한 화면에서 비교 가능해졌고, 이상 성과는 플래그로 자동 표시되어 마케팅팀이 집계 대신 전략 논의에 집중할 수 있는 환경을 만들었습니다.

가입 정보만으로 입금 확률 예측 — 머신러닝 기반 전환 예측 모델 개발

앤서스랩코리아 — GGPoker UK (GGPUK)
# LightGBM# SHAP 분석 # Snowflake + BigQuery# 데이터 누수 검증 # Salesforce MC 연동# 세그먼트 분석

가입자가 남긴 나이·기기·유입 채널·쿠폰 등 20개 정보로 "이 사람이 입금할 확률"을 미리 계산하는 예측 모델을 구축했습니다. 실제 입금 유저의 91%를 사전 식별하고, 어떤 요소가 확률을 올리고 내리는지 수치로 분해해 마케팅 예산 재배분 근거를 마련했습니다.

Situation
GGPUK는 2013~2026년 약 492,000명의 신규 가입자를 보유했으나, 가입자의 35%만이 첫 입금(FTD)으로 전환. 어떤 유저가 입금 가능성이 높은지 사전 식별 체계가 없어 마케팅 예산이 전환 가능성과 무관하게 집행되는 ROI 비효율 문제가 존재했습니다.
Task
가입 시점에 수집 가능한 정보만으로 FTD 전환 가능성을 예측하는 ML 모델 개발. Salesforce MC 및 광고 플랫폼 연동까지 실제 마케팅에 활용 가능한 형태로 구축. 임원진이 이해할 수 있는 보고서 작성.
Action
① 데이터 파이프라인 구축 — 3개 DB(Snowflake·BigQuery), 10개 테이블을 GGPASS_ID 공통 키로 조인하는 Wide Table 설계. ANONYMOUS_ID → USER_ID → GGPASS_ID 브릿지 구조 발굴.

② 데이터 누수(Leakage) 검증 — 40개 이상 후보 피처를 FTD=0/1 그룹 간 분포 비교(중앙값·nonzero 비율·mean ratio). FTD 발생 이후 생성되는 컬럼 전량 제거. 지역 피처(PAGES_VIEW)가 FTD=1에서만 99% 존재하는 구조적 한계 확인 및 제거.

③ 예측 모델 개발 (LightGBM) — "이 사람이 입금할 확률은 78%입니다"처럼 수치로 예측하는 모델을 구축했습니다. 나이·유입 채널·OS·쿠폰 사용 여부 등 가입 시점 20개 정보를 입력하면, 수천 번의 예측·수정 반복을 통해 점점 정확해지는 방식입니다. 3가지 버전의 모델을 비교해 "실제 입금 유저를 놓치지 않으면서 허수 예측도 줄이는" 균형 모델을 최종 선택했습니다.

④ SHAP 분석 — 예측 근거 투명화 — 모델이 "이 사람 78% 확률"이라고 예측했을 때, "왜?"를 숫자로 분해했습니다. 예시: 기본 확률 35%에서 출발 → 나이(45세) +12%p → iOS 기기 +8%p → FreeBonus 쿠폰 +6%p → 특정 BTAG 유입 +5%p → 합산 78%. 이 방식으로 age가 가장 영향력 있는 변수임을 수치로 증명하고, 고전환 세그먼트(BTAG 70%+, MEDIATROOPERS 81.6%, iOS 유저)를 마케팅 예산 재배분 근거로 제시했습니다.

⑤ 임원 보고서 자동 생성 — docx.js로 Word 문서 자동 생성. 데이터 구조·모델 성능·SHAP 시각화·세그먼트 분석·향후 계획을 포함하며, 비기술 독자도 이해할 수 있는 평어체로 작성.
Result
가입할 때 남긴 정보만으로 "이 사람이 입금할 확률"을 AUC 0.87 정확도로 예측 — 실제로 입금한 유저의 91%를 행동 데이터 없이 사전에 식별했습니다. 고전환 세그먼트를 수치로 정량화(상위 BTAG 파트너 FTD율 70%+, MEDIATROOPERS 채널 81.6%, iOS 유저 전반 우위)하여 마케팅 예산 재배분의 데이터 근거를 마련했습니다. 월간 신규 데이터 반영 및 AUC 자동 모니터링 파이프라인을 구축해 재현 가능한 운영 체계를 완성했습니다.
